SQL — Run in Supabase SQL Editor
-- Create users table
CREATE TABLE users (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  full_name TEXT NOT NULL,
  email TEXT UNIQUE NOT NULL,
  avatar_url TEXT,
  user_type TEXT DEFAULT 'resident',
  neighborhood TEXT,
  city TEXT,
  bio TEXT,
  badge_level INT DEFAULT 1,
  rep_points INT DEFAULT 0,
  is_newcomer BOOL DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Enable Row Level Security
ALTER TABLE users ENABLE ROW LEVEL SECURITY;

-- Policy: users can only read their own data
CREATE POLICY "Users read own data"
ON users FOR SELECT
USING (auth.uid() = id);

-- Create posts table
CREATE TABLE posts (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  post_type TEXT DEFAULT 'post',
  content TEXT NOT NULL,
  neighborhood TEXT,
  likes_count INT DEFAULT 0,
  is_flagged BOOL DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

Sign Up — replace button onclick in nearby-auth.html
async function handleSignup() {
  const email = document.getElementById('signupEmail').value
  const password = document.getElementById('signupPassword').value
  const name = document.getElementById('firstName').value

  const { data, error } = await supabase.auth.signUp({
    email: email,
    password: password,
    options: {
      data: { full_name: name }
    }
  })

  if (error) {
    alert(error.message)
  } else {
    // Insert user profile
    await supabase.from('users').insert({
      id: data.user.id,
      full_name: name,
      email: email,
      is_newcomer: true
    })
    window.location.href = 'nearby-onboarding.html'
  }
}
Sign In — replace login handler
async function handleLogin() {
  const email = document.getElementById('loginEmail').value
  const password = document.getElementById('loginPassword').value

  const { data, error } = await supabase.auth.signInWithPassword({
    email: email,
    password: password
  })

  if (error) {
    alert('Invalid email or password')
  } else {
    window.location.href = 'nearby-feed.html'
  }
}
Load Real Feed Posts
async function loadFeed() {
  const { data: posts, error } = await supabase
    .from('posts')
    .select('*, users(full_name, avatar_url, badge_level)')
    .order('created_at', { ascending: false })
    .limit(20)

  if (posts) {
    posts.forEach(post => renderPostCard(post))
  }
}

// Call on page load
loadFeed()

Go to Supabase → Authentication → Providers → Enable Google. Paste your Google Client ID and Secret.
Google OAuth Login
async function handleGoogle() {
  const { data, error } = await supabase.auth.signInWithOAuth({
    provider: 'google',
    options: {
      redirectTo: window.location.origin + '/nearby-feed.html'
    }
  })
}
Protect Pages — add to every page top
// Check user is logged in
const { data: { session } } = await supabase.auth.getSession()

if (!session) {

}

🔐 Security Implementation Checklist
🔒
HTTPS everywhere
Vercel gives you SSL/HTTPS automatically. All data between users and the server is encrypted in transit.
Auto ✓
🛡️
Row Level Security (RLS)
Supabase ensures users can only access their own data. No user can see another user's private information.
SQL Done ✓
🔑
JWT Authentication
Supabase uses secure JSON Web Tokens for all sessions. Tokens expire and refresh automatically.
Auto ✓
📍
Location encryption
Never store exact GPS. Store only neighborhood strings. Only show city or area name on profiles.
Build →
💬
Message encryption
Use TweetNaCl.js library to encrypt messages client-side before sending to Supabase. Only the two users can decrypt.
Build →
🚦
Rate limiting
Supabase has built-in rate limiting on auth endpoints. Add Vercel Edge Middleware for API rate limiting.
Auto ✓
🧹
Input sanitization
All user inputs must be sanitized before inserting into database to prevent SQL injection and XSS attacks.
